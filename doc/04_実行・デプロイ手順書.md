## 1. 必要な環境（事前準備）

### 1.1 作業PC（ビルド環境）に必要なもの

| ソフトウェア | バージョン | 確認コマンド | ダウンロード先 |
|---|---|---|---|
| **Java JDK** | 17以上 | `java -version` | https://adoptium.net/ja/ |
| **Apache Maven** | 3.6以上 | `mvn -version` | https://maven.apache.org/download.cgi |
| **Git** | 任意 | `git --version` | https://git-scm.com/ |

> **💡 ヒント**: Java が入っているか確認するには、コマンドプロンプト（Windowsキー + R → `cmd`）で `java -version` と入力してください。バージョン番号が表示されればOKです。

| ソフトウェア | 確認方法 |

| Apache Tomcat 9.0.16 | `/opt/apache-tomcat/apache-tomcat-9.0.16/bin/version.sh` |
| MySQL 8.x | `mysql --version` |

## 2. 初回セットアップ手順

### 2.1 MySQLデータベースの準備

サーバーにSSH接続し、以下を実行します。

```bash
# MySQLにログイン
mysql -u root -p
# パスワード: Abcc123.# を入力
```

MySQLに入ったら、以下のSQLを実行します：

```sql
-- データベースが存在しない場合は作成
CREATE DATABASE IF NOT EXISTS asopoketool
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_general_ci;

-- 確認
SHOW DATABASES;
-- 「asopoketool」が表示されればOK

-- MySQLを終了
exit;
```

> **✅ ポイント**: テーブルの作成は、アプリを初回起動したときに **自動で行われます**（Flyway という仕組みを使っています）。手動でSQLを実行する必要はありません。

### 2.2 BGMアップロードディレクトリの作成（任意）

```bash
# BGMファイルの保存先を作成
sudo mkdir -p /opt/asopoketool/bgm
sudo chown tomcat:tomcat /opt/asopoketool/bgm
sudo chmod 755 /opt/asopoketool/bgm
```

## 3. ビルド（WARファイルの作成）

### 3.1 プロジェクトのディレクトリに移動

Windowsのコマンドプロンプトで：

```cmd
cd c:\repo\asopoketool
```

### 3.2 ビルド実行

```cmd
mvn clean package -DskipTests
```

> **💡 説明**:
> - `clean`: 前回のビルド結果をきれいに削除します
> - `package`: ソースコードをコンパイルして WARファイルを作成します
> - `-DskipTests`: テストをスキップして速くビルドします

**初回ビルド時は、インターネットからライブラリをダウンロードするため5〜10分かかる場合があります。**

### 3.3 ビルド成功の確認

以下のメッセージが表示されればOKです：

```
[INFO] BUILD SUCCESS
```

生成されたWARファイルを確認します：

```cmd
dir target\asopoketool.war
```

`asopoketool.war` というファイルが存在すればビルド成功です。

## 4. サーバーへのデプロイ

### 4.1 WARファイルをサーバーに転送

**方法A: scpコマンド（Linuxコマンドライン）**

```bash
scp target/asopoketool.war ユーザー名@サーバーIP:/tmp/asopoketool.war
```

**方法B: WinSCP（Windows GUI）**
1. WinSCP をダウンロード・起動 (https://winscp.net/eng/download.php)
2. サーバーに接続
3. `target\asopoketool.war` を `/tmp/` にドラッグ＆ドロップ

### 4.2 Tomcat に配置

サーバー上で実行：

```bash
# Tomcatの停止
sudo /opt/apache-tomcat/apache-tomcat-9.0.16/bin/shutdown.sh

# 少し待つ（5秒程度）
sleep 5

# 古いデプロイ済みファイルを削除（2回目以降のデプロイ時）
sudo rm -rf /opt/apache-tomcat/apache-tomcat-9.0.16/webapps/asopoketool
sudo rm -f  /opt/apache-tomcat/apache-tomcat-9.0.16/webapps/asopoketool.war

# 新しいWARファイルを配置
sudo cp /tmp/asopoketool.war /opt/apache-tomcat/apache-tomcat-9.0.16/webapps/

# ファイルの権限を設定
sudo chown tomcat:tomcat /opt/apache-tomcat/apache-tomcat-9.0.16/webapps/asopoketool.war

# Tomcatを起動
sudo /opt/apache-tomcat/apache-tomcat-9.0.16/bin/startup.sh
```

### 4.3 起動ログの確認

```bash
# ログをリアルタイムで確認（Ctrl+C で止める）
tail -f /opt/apache-tomcat/apache-tomcat-9.0.16/logs/catalina.out
```

以下のメッセージが表示されれば起動成功です：

```
Started AsoPokeToolApplication in X.XXX seconds
```

> **⚠️ 注意**: 初回起動時はデータベースのテーブルを自動作成するため、通常より時間がかかります（30秒〜1分程度）。

## 8. よくあるトラブルと対処法

### ❌ 問題: ブラウザで「503 Service Unavailable」が表示される

**原因**: Tomcat が起動していない

**対処**:
```bash
# Tomcatの状態確認
ps aux | grep tomcat

# 起動していなければ起動
sudo /opt/apache-tomcat/apache-tomcat-9.0.16/bin/startup.sh

# ログを確認
tail -100 /opt/apache-tomcat/apache-tomcat-9.0.16/logs/catalina.out
```

---

### ❌ 問題: 「データベース接続エラー」が表示される

**原因**: MySQLが起動していない、またはパスワードが違う

**対処**:
```bash
# MySQLの状態確認
sudo systemctl status mysql

# 起動していなければ起動
sudo systemctl start mysql

# 接続テスト
mysql -u root -p'Abcc123.#' -e "SHOW DATABASES;"
```

---

### ❌ 問題: ビルドで「BUILD FAILURE」になる

**確認事項**:
1. Java 17以上がインストールされているか: `java -version`
2. Maven がインストールされているか: `mvn -version`
3. インターネットに接続できるか（初回ビルド時はダウンロードが必要）
4. `pom.xml` のあるディレクトリでコマンドを実行しているか

---

### ❌ 問題: QRコードが表示されない

**確認事項**:
- ブラウザのカメラ許可設定を確認
- HTTPSで接続しているか（カメラ使用にはHTTPS推奨）

---

### ❌ 問題: BGMがアップロードできない

**原因**: アップロードディレクトリの権限不足

**対処**:
```bash
sudo mkdir -p /opt/asopoketool/bgm
sudo chown -R tomcat:tomcat /opt/asopoketool
sudo chmod -R 755 /opt/asopoketool
```

---

### 📋 ログファイルの場所

| ログの種類 | ファイルパス |
|---|---|
| Tomcat 起動ログ | `/opt/apache-tomcat/apache-tomcat-9.0.16/logs/catalina.out` |

## 9. 大会運営の操作フロー（クイックガイド）

```
1. 管理者でログイン → 大会を作成
   /admin/login → /admin/tournament/create

2. 参加者がエントリー（当日まで受付）
   / → 大会を選択 → エントリー

3. 当日チェックイン
   参加者: QRコードを表示 (/checkin/qr/{entryId})
   管理者: QRコードをスキャン (/admin/tournament/{id}/checkin)

4. スイスドロー開始
   /admin/tournament/{id} → 「大会開始」ボタン

5. ラウンドを進める
   マッチング生成 → 対戦 → 結果登録 → 次ラウンド
   /admin/tournament/{id}/matching → 「マッチング生成」

6. 対戦結果の登録
   参加者: /tournament/{id}/result/{matchId}
   管理者: /admin/tournament/{id}/matching（管理者用ボタン）

7. 全ラウンド終了後、決勝トーナメント
   /admin/tournament/{id}/bracket → 「トーナメント生成」

8. 大会終了・ポイント集計
   /admin/tournament/{id} → 「大会終了」ボタン
```

