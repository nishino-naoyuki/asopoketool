## 2. 初回セットアップ手順（MySQL・Apache）

### 2.1 MySQLデータベースの準備

サーバーにSSH接続し、MySQLにログインして新しく `asopoketool` という名前のデータベース（スキーマ）を作成します。

```bash
# MySQLにログイン
mysql -u root -p
# パスワードを入力
```

MySQLに入ったら、以下のSQLを実行します：

```sql
-- データベースがまだ存在しない場合は作成
CREATE DATABASE IF NOT EXISTS asopoketool
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_general_ci;

-- 作成されたか確認
SHOW DATABASES;
-- リストに「asopoketool」が表示されれば成功です

-- 接続確認後、MySQLを終了
exit;
```

> **✅ ポイント**: テーブルの作成は、アプリを初回起動したときに **自動で行われます**（Flyway という仕組みを使っています）。手動でSQLを実行する必要はありません。また、既存の他のデータベースと干渉することもありません。

### 2.2 Apache (ssl.conf) の転送設定の追加

`https://サーバーのIPまたはドメイン/asopoketool` というURLでアプリに接続できるように、Apache のリバースプロキシ設定を追加します。

1. 設定ファイル（`/etc/httpd/conf.d/ssl.conf`）をテキストエディタで開きます。
   ```bash
   sudo vi /etc/httpd/conf.d/ssl.conf
   ```

2. `<VirtualHost _default_:443>` ブロック内に記述されている、既存の Javaアプリ転送設定（例: `/azcafe` の設定）の直下に、以下の **AJP転送設定（3行）** を追記します。

   ```apache
   # =================================================================
   # ASOPokeTool アプリケーション設定 (追記)
   # =================================================================
   RewriteCond %{REQUEST_URI} ^/asopoketool(/.*)?$
   RewriteRule ^/asopoketool/(.*)$ ajp://localhost:8009/asopoketool/$1 [P,L]
   ```

3. **Apacheの設定テストと再読み込み**
   記述に文法エラーがないかチェックし、問題なければ Apache の設定を再読み込みします。
   ```bash
   # 設定のチェック (Syntax OK と出れば成功)
   sudo apachectl configtest

   # 設定の再読み込み (サービスを停止させずに即時反映)
   sudo systemctl reload httpd
   ```
   > [!TIP]
   > `systemctl reload` は、稼働中の他のWebアプリケーションやPythonプログラムを停止・再起動させずに設定だけを優しく再ロードするため、本番稼働中のサーバーでも安全に実行できます。

## 3. ビルド（WARファイルの作成）

### 3.1 プロジェクトのディレクトリに移動

Windowsのコマンドプロンプト（または PowerShell）でプロジェクトのルートに移動します：

```cmd
cd c:\repo\asopoketool
```

### 3.2 ビルド実行

PCにMavenがインストールされていない場合は、以下のコマンドで同梱のスクリプト等を使ってビルドします：

```cmd
powershell -ExecutionPolicy Bypass -File scratch/build-war.ps1
```

※PC環境にMavenやJavaへのパスが通っている場合は、直接以下を実行しても構いません：
```cmd
mvn clean package -DskipTests
```

### 3.3 ビルド成功の確認

以下のメッセージが表示されれば成功です：

```
[INFO] BUILD SUCCESS
```

生成されたWARファイルを確認します：
`target\asopoketool.war` というファイルが存在すればビルド完了です。

## 4. サーバーへのデプロイ

### 4.1 WARファイルをサーバーに転送

WinSCP や FileZilla などのSFTPツールを使い、ビルドされた `target/asopoketool.war` をサーバーの `/tmp/` フォルダへアップロードします。

### 4.2 Tomcat に配置（ホットデプロイ）

サーバーにSSH接続し、Tomcat の配備フォルダ（`webapps`）へWARファイルを配置します。

```bash
# 既存の古いWARファイルと展開フォルダを削除（2回目以降のデプロイ時）
sudo rm -rf /usr/share/tomcat/webapps/asopoketool
sudo rm -f  /usr/share/tomcat/webapps/asopoketool.war

# 新しいWARファイルをTomcatのwebappsフォルダへコピー
sudo cp /tmp/asopoketool.war /usr/share/tomcat/webapps/

# コピーしたファイルの所有者を tomcat ユーザーに設定
sudo chown tomcat:tomcat /usr/share/tomcat/webapps/asopoketool.war
```
> [!IMPORTANT]
> Tomcat が起動していれば、WARファイルを配置した数秒〜数十秒後に自動で検出されて `/asopoketool` フォルダが作成され、アプリが自動起動（ホットデプロイ）します。他システムを一時停止する必要はありません。

### 4.3 起動ログの確認

アプリが正常に起動しているか、Tomcat のログを確認します。
```bash
tail -f /var/log/tomcat/catalina.out
# (Ctrl + C で表示を終了できます)
```

ログに以下のように表示されれば起動成功です：
```
Started AsoPokeToolApplication in X.XXX seconds
```

## 8. よくあるトラブルと対処法

### ❌ 問題: アップロード画像やBGMの保存時にエラーになる
* **原因**: 画像やBGMなどのアップロード用フォルダ（`./data/images/` など）を作成・書き込みするための権限が、Tomcat実行ユーザー（`tomcat`）に不足している可能性があります。
* **対処**: Tomcatの実行ユーザーがアプリ展開ディレクトリ、もしくは作業フォルダに対して正しくパーミッション（書き込み権限）を持っているか確認し、必要に応じて権限を付与してください。
  ```bash
  # デプロイ先ディレクトリの所有者を tomcat にする例
  sudo chown -R tomcat:tomcat /usr/share/tomcat/webapps/asopoketool
  ```

### ❌ 問題: ブラウザでアクセスすると「503 Service Unavailable」になる
* **原因**: Tomcat自体が起動していない、あるいはAJPポート（8009）が有効になっていません。
* **対処**:
  ```bash
  # Tomcatのプロセスが動いているか確認
  ps aux | grep tomcat
  
  # 起動していない場合はTomcatを起動
  sudo systemctl start tomcat
  ```

## 9. 大会運営の操作フロー（クイックガイド）

```
1. 管理者でログイン → 大会を作成
   /admin/login → /admin/tournament/create

2. 参加者がエントリー（当日まで受付）
   / → 大会を選択 → エントリー

3. 当日チェックイン
   参加者: QRコードを表示 (/checkin/qr/{entryId})
   管理者: QRコードをスキャン (/admin/tournament/{id}/checkin)

4. スイスドロー開始
   /admin/tournament/{id} → 「大会開始」ボタン

5. ラウンドを進める
   マッチング生成 → 対戦 → 結果登録 → 次ラウンド
   /admin/tournament/{id}/matching → 「マッチング生成」

6. 対戦結果の登録
   参加者: /tournament/{id}/result/{matchId}
   管理者: /admin/tournament/{id}/matching（管理者用ボタン）

7. 全ラウンド終了後、決勝トーナメント
   /admin/tournament/{id}/bracket → 「トーナメント生成」

8. 大会終了・ポイント集計
   /admin/tournament/{id} → 「大会終了」ボタン
```

## 10. ファイル構成まとめ

```
/usr/share/tomcat/webapps/
├── asopoketool.war        ← デプロイしたWARファイル
└── asopoketool/           ← Tomcatが自動展開したフォルダ
    └── data/
        ├── images/        ← ユーザーアイコンなどのアップロード画像保存先
        └── bgm/           ← アップロードされたBGMファイル保存先
```
