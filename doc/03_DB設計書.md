## 3. テーブル定義

### 3.1 tournament（大会テーブル）

| カラム名 | 型 | NOT NULL | デフォルト | 説明 |
|---|---|---|---|---|
| `id` | BIGINT UNSIGNED | YES | AUTO_INCREMENT | 大会ID（PK） |
| `name` | VARCHAR(100) | YES | - | 大会名 |
| `held_date` | DATE | YES | - | 開催日 |
| `capacity` | INT | YES | 32 | 定員 |
| `total_rounds` | INT | YES | 5 | スイスドロー総ラウンド数 |
| `current_round` | INT | YES | 0 | 現在のラウンド（0=未開始） |
| `win_point` | INT | YES | 3 | 勝利ポイント |
| `lose_point` | INT | YES | 0 | 敗北ポイント |
| `participation_point` | INT | YES | 1 | 参加ポイント |
| `bracket_group_size` | INT | YES | 8 | 決勝T 1グループの人数 |
| `status` | ENUM('ENTRY','IN_PROGRESS','BRACKET','FINISHED') | YES | 'ENTRY' | 大会ステータス |
| `created_at` | DATETIME | YES | CURRENT_TIMESTAMP | 作成日時 |
| `updated_at` | DATETIME | YES | CURRENT_TIMESTAMP ON UPDATE | 更新日時 |

```sql
CREATE TABLE tournament (
    id                   BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    name                 VARCHAR(100) NOT NULL,
    held_date            DATE NOT NULL,
    capacity             INT NOT NULL DEFAULT 32,
    total_rounds         INT NOT NULL DEFAULT 5,
    current_round        INT NOT NULL DEFAULT 0,
    win_point            INT NOT NULL DEFAULT 3,
    lose_point           INT NOT NULL DEFAULT 0,
    participation_point  INT NOT NULL DEFAULT 1,
    bracket_group_size   INT NOT NULL DEFAULT 8,
    status               ENUM('ENTRY','IN_PROGRESS','BRACKET','FINISHED') NOT NULL DEFAULT 'ENTRY',
    created_at           DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at           DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3.2 entry（エントリーテーブル） ※アカウント機能追加により変更

| カラム名 | 型 | NOT NULL | デフォルト | 説明 |
|---|---|---|---|---|
| `id` | BIGINT UNSIGNED | YES | AUTO_INCREMENT | エントリーID（PK） |
| `tournament_id` | BIGINT UNSIGNED | YES | - | 大会ID（FK: tournament.id） |
| `player_name` | VARCHAR(50) | YES | - | エントリー名（大会ごとに任意の名前） |
| **`account_id`** | **BIGINT UNSIGNED** | **NO** | **NULL** | **アカウントID（FK: player_account.id、ゲストはNULL）** |
| `session_token` | VARCHAR(255) | YES | - | エントリー識別用セッショントークン（ゲスト用。会員も保持） |
| `qr_token` | VARCHAR(512) | YES | - | QRコード用トークン（HMAC含む） |
| `checkin_flg` | TINYINT(1) | YES | 0 | チェックインフラグ（0:未/1:済） |
| `checkin_at` | DATETIME | NO | NULL | チェックイン日時 |
| `dropout_flg` | TINYINT(1) | YES | 0 | 棄権フラグ（0:参加中/1:棄権） |
| `dropout_at` | DATETIME | NO | NULL | 棄権日時 |
| `manual_entry_flg` | TINYINT(1) | YES | 0 | 手動登録フラグ（飛び込み参加者） |
| `created_at` | DATETIME | YES | CURRENT_TIMESTAMP | 登録日時 |
| `updated_at` | DATETIME | YES | CURRENT_TIMESTAMP ON UPDATE | 更新日時 |

> **設計ポイント**:
> - `player_name` は大会ごとのエントリー名。アカウントの `display_name` とは独立して管理する。
> - `account_id` を持つことで同一人物が大会ごとに異なる名前を使っていても正しく同一視できる。
> - ゲスト（未登録）エントリーは `account_id = NULL`、`session_token` で識別する。

```sql
CREATE TABLE entry (
    id               BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    tournament_id    BIGINT UNSIGNED NOT NULL,
    player_name      VARCHAR(50) NOT NULL,
    account_id       BIGINT UNSIGNED DEFAULT NULL,
    session_token    VARCHAR(255) NOT NULL,
    qr_token         VARCHAR(512) NOT NULL,
    checkin_flg      TINYINT(1) NOT NULL DEFAULT 0,
    checkin_at       DATETIME DEFAULT NULL,
    dropout_flg      TINYINT(1) NOT NULL DEFAULT 0,
    dropout_at       DATETIME DEFAULT NULL,
    manual_entry_flg TINYINT(1) NOT NULL DEFAULT 0,
    created_at       DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at       DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (tournament_id) REFERENCES tournament(id) ON DELETE CASCADE,
    FOREIGN KEY (account_id)    REFERENCES player_account(id) ON DELETE SET NULL,
    INDEX idx_tournament_session (tournament_id, session_token),
    INDEX idx_tournament_account (tournament_id, account_id),
    UNIQUE KEY uk_qr_token (qr_token)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

| `display_name` | VARCHAR(50) | YES | - | 表示名（エントリーのデフォルト名。変更可能） |
| `password_hash` | VARCHAR(255) | YES | - | パスワードハッシュ（BCrypt） |
| `icon_path` | VARCHAR(500) | NO | NULL | ユーザーアイコンの画像ファイルパス（NEW） |
| `created_at` | DATETIME | YES | CURRENT_TIMESTAMP | 登録日時 |
| `updated_at` | DATETIME | YES | CURRENT_TIMESTAMP ON UPDATE | 更新日時 |

> **設計ポイント**:
> - `display_name` はアカウントの表示名。大会のエントリー名（`entry.player_name`）とは独立。
> - 同一人物が複数の大会で異なるエントリー名を使う場合も、`account_id` で同一人物として識別できる。
> - `display_name` はデフォルトエントリー名として U-01 登録時・即エントリー時に使用する。

```sql
CREATE TABLE player_account (
    id            BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    display_name  VARCHAR(50) NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    icon_path     VARCHAR(500) DEFAULT NULL,
    created_at    DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at    DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_display_name (display_name)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3.15 player_account_token（自動ログイントークンテーブル） 【NEW】

| カラム名 | 型 | NOT NULL | デフォルト | 説明 |
|---|---|---|---|---|
| `id` | BIGINT UNSIGNED | YES | AUTO_INCREMENT | トークンID（PK） |
| `account_id` | BIGINT UNSIGNED | YES | - | アカウントID（FK: player_account.id） |
| `token` | VARCHAR(512) | YES | - | 自動ログイントークン（UUID v4、HMAC） |
| `expire_at` | DATETIME | YES | - | トークン有効期限（発行から120日） |
| `created_at` | DATETIME | YES | CURRENT_TIMESTAMP | 発行日時 |
| `last_used_at` | DATETIME | NO | NULL | 最終使用日時 |

> **設計ポイント**:
> - トークンは Cookie（`APT_TOKEN`、`HttpOnly`、`SameSite=Lax`）にセットする。
> - Cookie の値と DB の `token` を照合し、`expire_at` が現在日時より未来なら自動ログイン。
> - ログアウト時は該当レコードを削除（または `expire_at` を過去に設定）してトークン無効化。
> - 同一アカウントで複数デバイスログインを許容するためアカウント1対多でトークンを管理。
> - 古いトークン（期限切れ）はバッチで定期削除する。

```sql
CREATE TABLE player_account_token (
    id           BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    account_id   BIGINT UNSIGNED NOT NULL,
    token        VARCHAR(512) NOT NULL,
    expire_at    DATETIME NOT NULL,
    created_at   DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    last_used_at DATETIME DEFAULT NULL,
    FOREIGN KEY (account_id) REFERENCES player_account(id) ON DELETE CASCADE,
    UNIQUE KEY uk_token (token),
    INDEX idx_account_id (account_id),
    INDEX idx_expire_at  (expire_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 5. データフロー

### 5.1 大会の状態遷移

```
ENTRY（エントリー受付中）
    |
    | チェックイン締め切り・大会開始
    v
IN_PROGRESS（スイスドロー進行中）
    | current_round が 1 → total_rounds に増加
    |
    | スイスドロー全ラウンド完了
    v
BRACKET（決勝トーナメント）
    |
    | 決勝トーナメント完了
    v
FINISHED（大会終了・ポイント計算）
```

### 5.2 アカウント自動ログインフロー

```
ブラウザアクセス
    |
    +-- Cookie "APT_TOKEN" 存在しない → ゲスト状態（ヘッダーにログイン・登録ボタン）
    |
    +-- Cookie "APT_TOKEN" 存在する
            |
            +-- player_account_token WHERE token = ? AND expire_at > NOW()
            |       |
            |       +-- レコードなし → トークン無効 → Cookie削除 → ゲスト状態
            |       |
            |       +-- レコードあり → account_id を取得 → player_account を JOIN
            |               → last_used_at を現在時刻に更新
            |               → ログイン状態（ヘッダーに名前表示）

ログアウト
    +-- player_account_token WHERE token = ? を DELETE
    +-- Cookie "APT_TOKEN" を削除
    +-- ゲスト状態へ
```

### 5.3 エントリーフロー（アカウント会員）

```
[即エントリー]
    → entry.player_name = player_account.display_name
    → entry.account_id  = ログイン中の account_id
    → entry.session_token = 現在のセッション

[エントリー名を指定してエントリー]
    → entry.player_name = 入力された任意の名前（アカウント名と異なってもOK）
    → entry.account_id  = ログイン中の account_id
    → entry.session_token = 現在のセッション

[ゲストエントリー]
    → entry.player_name = 入力された名前
    → entry.account_id  = NULL
    → entry.session_token = 現在のセッション
```

### 5.4 スイスドロー処理における対戦履歴参照クエリ（概要）

```sql
-- 同一大会内で対戦済みのペアを取得（同じ人と当たらないようにするため）
SELECT mg.player1_entry_id, mg.player2_entry_id
FROM match_game mg
JOIN tournament_round tr ON mg.round_id = tr.id
WHERE tr.tournament_id = :tournamentId;
```

### 5.5 ランキング計算クエリ（概要）

```sql
-- スイスドロー勝ち数集計
SELECT
    e.id            AS entry_id,
    e.player_name,
    e.account_id,
    SUM(CASE WHEN mr.winner_entry_id = e.id THEN 1 ELSE 0 END) AS wins,
    COUNT(mr.id)    AS matches_played
FROM entry e
LEFT JOIN match_game mg
    ON (mg.player1_entry_id = e.id OR mg.player2_entry_id = e.id)
LEFT JOIN match_result mr ON mr.match_id = mg.id
JOIN tournament_round tr  ON mg.round_id = tr.id
WHERE e.tournament_id     = :tournamentId
  AND tr.tournament_id    = :tournamentId
GROUP BY e.id, e.player_name, e.account_id;

-- オポネント率はアプリケーション側（Java）で計算
-- 各対戦相手の勝率を計算し、0.3未満は0.3として平均を取る
```

### 5.6 マイページ（過去大会履歴）クエリ（概要）

```sql
-- アカウントIDに紐づく過去の大会エントリー一覧
SELECT
    t.name         AS tournament_name,
    t.held_date,
    e.player_name  AS entry_name,
    ph.final_rank,
    ph.total_point,
    (SELECT COUNT(*) FROM match_result mr2
     JOIN match_game mg2 ON mr2.match_id = mg2.id
     JOIN tournament_round tr2 ON mg2.round_id = tr2.id
     WHERE tr2.tournament_id = t.id AND mr2.winner_entry_id = e.id) AS wins,
    (SELECT COUNT(*) FROM match_game mg3
     JOIN tournament_round tr3 ON mg3.round_id = tr3.id
     WHERE tr3.tournament_id = t.id
       AND (mg3.player1_entry_id = e.id OR mg3.player2_entry_id = e.id)) AS total_matches
FROM entry e
JOIN tournament t ON e.tournament_id = t.id
LEFT JOIN point_history ph
    ON ph.entry_id = e.id AND ph.tournament_id = t.id
WHERE e.account_id = :accountId
ORDER BY t.held_date DESC;
```
