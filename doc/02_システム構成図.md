## 2. 技術スタック

### 2.1 フロントエンド（モバイルアプリ）

| カテゴリ | 技術 | 理由 |
|----------|------|------|
| フレームワーク | Flutter | iOS/Android同時開発、高パフォーマンス |
| 状態管理 | Riverpod | シンプルで型安全 |
| ローカルDB | SQLite | オフライン対応 |
| QRスキャン | mobile_scanner | 高速・軽量 |

### 2.2 バックエンド

| カテゴリ | 技術 | 理由 |
|----------|------|------|
| 言語 | TypeScript | 型安全、フロントと共通 |
| フレームワーク | NestJS | エンタープライズ向け、DI対応 |
| ORM | Prisma | 型安全、マイグレーション管理 |
| 認証 | JWT + Refresh Token | ステートレス、5日間キャッシュ対応 |

### 2.3 インフラストラクチャ

| カテゴリ | 技術 | 理由 |
|----------|------|------|
| クラウド | AWS | 実績・信頼性 |
| コンテナ | ECS Fargate | サーバーレス運用 |
| DB | RDS PostgreSQL | 信頼性・ACID準拠 |
| キャッシュ | ElastiCache Redis | 高速セッション管理 |
| CDN | CloudFront | 画像配信高速化 |
| ストレージ | S3 | グッズン画像保存 |

## 3. サービス構成詳細

### 3.1 認証サービス (Auth Service)
```mermaid
sequenceDiagram
    participant App as モバイルアプリ
    participant Auth as 認証サービス
    participant Redis as Redis
    participant DB as PostgreSQL

    App->>Auth: ログイン要求
    Auth->>DB: ユーザー認証
    DB-->>Auth: ユーザー情報
    Auth->>Auth: JWT発行（5日有効）
    Auth->>Redis: RefreshToken保存
    Auth-->>App: AccessToken + RefreshToken
```

### 3.2 バトルサービス (Battle Service)
```mermaid
sequenceDiagram
    participant App as モバイルアプリ
    participant Battle as バトルサービス
    participant Quiz as クイズサービス
    participant DB as PostgreSQL

    App->>Battle: バトル開始（QRデータ）
    Battle->>DB: 敵情報・地形取得
    Battle-->>App: バトル情報

    loop クイズターン
        App->>Battle: ジャンル選択
        Battle->>Quiz: クイズ取得要求
        Quiz->>DB: クイズ取得（難易度別）
        Quiz-->>Battle: クイズデータ
        Battle-->>App: クイズ出題
        App->>Battle: 回答送信
        Battle->>Battle: ダメージ計算
        Battle-->>App: 結果（ダメージ/敵攻撃）
    end

    Battle->>DB: バトル結果保存
    Battle-->>App: 報酬（スキルポイント）
```

### 3.3 ガチャサービス (Gacha Service)
```mermaid
sequenceDiagram
    participant App as モバイルアプリ
    participant Gacha as ガチャサービス
    participant DB as PostgreSQL

    App->>Gacha: ガチャ実行要求
    Gacha->>DB: コイン残高確認
    alt コイン不足
        Gacha-->>App: エラー（コイン不足）
    else コイン十分
        Gacha->>Gacha: 抽選処理
        Gacha->>DB: コイン消費
        Gacha->>DB: グッズン付与
        Gacha-->>App: 獲得グッズン情報
    end
```

## 5. セキュリティ設計

### 5.1 認証・認可
```mermaid
flowchart LR
    A[ログイン] --> B[JWT AccessToken発行<br/>有効期限: 1時間]
    B --> C[API呼び出し]
    C --> D{Token有効?}
    D -->|Yes| E[API処理実行]
    D -->|No| F[RefreshToken確認]
    F --> G{RefreshToken有効?<br/>5日以内}
    G -->|Yes| H[新AccessToken発行]
    G -->|No| I[再ログイン要求]
    H --> C
```

### 5.2 セキュリティ対策

| 対策 | 説明 |
|------|------|
| HTTPS強制 | 全通信をTLS1.3で暗号化 |
| パスワードハッシュ | bcryptでソルト付きハッシュ |
| レート制限 | API呼び出し回数制限 |
| QR検証 | 署名付きQRコードで改ざん防止 |
| SQLインジェクション | ORMによるパラメータバインド |
